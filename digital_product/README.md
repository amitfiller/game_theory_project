# מוצר דיגיטלי: "AI בלי להיות טכנולוגי" 🚀

מוצר דיגיטלי מוכן למכירה לפי המתכון: איבוק → Canva/PDF → Stan Store (₪49) → רילס + ManyChat.

**הנישה:** מדריכי "AI ללא-טכניים" — הקטגוריה הצומחת ביותר במוצרים דיגיטליים ב-2026.

## מה יש כאן

```
digital_product/
├── README.md                      ← אתם כאן
├── ebook/
│   ├── ai-guide-hebrew.md         ← תוכן המדריך המלא (מקור, לעריכה)
│   ├── ai-guide-hebrew.pdf        ← המוצר המוכן למכירה (20 עמ', מעוצב, RTL)
│   └── build_pdf.py               ← סקריפט שבונה את ה-PDF מה-markdown
└── marketing/
    ├── stan-store-copy.md         ← קופי מוכן לדף המכירה + FAQ + ביו
    ├── reels-scripts.md           ← 10 תסריטי רילס + לוח פרסום חודשי
    ├── setup-guide.md             ← הקמת Stan Store + ManyChat צעד-אחר-צעד
    └── launch-checklist.md        ← צ'ק-ליסט השקה של 7 ימים
```

## מה במדריך עצמו (המוצר)

8 פרקים, 20 עמודים: היכרות עם הכלים החינמיים, 5 כללי זהב לכתיבת פרומפטים + תבנית אוניברסלית, שימושים לחיי היומיום ולעסק, 5 מסלולי הכנסה אמיתיים, **50 פרומפטים מוכנים בעברית**, וצ'ק-ליסט לשבוע הראשון.

## איך מתחילים (סדר פעולות)

1. **קראו את ה-PDF** — זה המוצר שלכם, כדאי להכיר אותו בעל-פה
2. **מומלץ: הוסיפו נגיעה אישית** — ערכו את `ebook/ai-guide-hebrew.md` (סיפור שלכם בפתיחה, דוגמה מהחיים שלכם), והריצו `python3 ebook/build_pdf.py` לבנייה מחדש
3. **עקבו אחרי `marketing/launch-checklist.md`** — 7 ימים מהיום למכירה ראשונה
4. ההקמה הטכנית מפורטת ב-`marketing/setup-guide.md`, הקופי מוכן ב-`marketing/stan-store-copy.md`, והתוכן לרילס ב-`marketing/reels-scripts.md`

## עריכה ובנייה מחדש של ה-PDF

```bash
pip install weasyprint markdown pypdfium2
python3 digital_product/ebook/build_pdf.py
```

הסקריפט משתמש בפונטים Heebo/Assistant/Noto Emoji (מותקנים ב-`/home/user/fonts` — בסביבה אחרת הורידו אותם מ-Google Fonts ועדכנו את הנתיב בראש הסקריפט).

## תזכורות חשובות

- מילת המפתח לאוטומציה בכל התסריטים: **"מדריך"**
- נקודת איזון: ~4 מכירות בחודש מכסות את עלויות הפלטפורמות
- הכנסות = חייבות דיווח. עוסק פטור + שיחה עם רו"ח לפני ההשקה
- אל תשפטו תוצאות לפני 30 יום של פרסום עקבי
